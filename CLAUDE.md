# CLAUDE.md — nps-frontend

## Project Overview

Flutter client that collects NPS feedback (phase 1) and, eventually, also
files Open311 service requests against geographic features picked from a
pygeoapi instance (phase 2+). Despite the `nps-` prefix the app is the
unified civic-interaction frontend for the whole `ruohomaki.fi` family of
APIs.

System-level architecture (topology, shared schemas, phase plan) lives in
[`docs/ARCHITECTURE.md`](docs/ARCHITECTURE.md). This file documents the
frontend repository itself.

## Architecture

- **Framework:** Flutter 3.22+ / Dart 3.4+
- **Targets:** iOS, Android, and macOS desktop (macOS is the demo platform)
- **State:** plain `StatefulWidget` for now — no Riverpod/Bloc until the
  screen graph grows past two screens
- **Network:** `package:http` directly; no Dio/Chopper layer
- **Config:** every runtime knob comes from `--dart-define`. No `.env`
  files, no secrets in source. See [Configuration](#configuration).

## Conventions

- **File size:** keep Dart files under ~250 lines. Split widgets aggressively.
- **`hide Feedback`:** `lib/screens/nps_screen.dart` does
  `import 'package:flutter/material.dart' hide Feedback;` because Flutter's
  material library exports a `Feedback` class that collides with our model.
  Every screen file that touches the model must do the same.
- **No emojis in source files** unless the user explicitly asks for them.
- **No `print` calls.** Use `debugPrint` in dev paths only; production code
  surfaces errors via UI (snackbar/banner/error state).
- **No magic strings.** Service IDs, asset paths, dart-define keys go
  through `AppConfig` or named constants.
- **Tests:** model + business logic must have unit tests. Widget tests
  optional but encouraged for non-trivial screens.

## Configuration

All runtime config via `--dart-define`. Reader: `lib/config.dart`.

| Var | Default | Description |
|---|---|---|
| `API_BASE_URL` | `https://api.ruohomaki.fi/nps` | nps-api base URL, no trailing slash |
| `API_KEY` | (empty) | Value sent as `X-API-Key` header. Empty = header omitted (matches nps-api with `API_KEYS` unset) |
| `APP_ID` | `nps-frontend-demo` | Value sent as the `app` field |

Planned for phase 2:

| Var | Default | Description |
|---|---|---|
| `GEO_BASE_URL` | `https://api.ruohomaki.fi/geo` | pygeoapi base URL |
| `OPEN311_BASE_URL` | `https://api.ruohomaki.fi/open311` | open311-to-Go base URL |
| `DEMO_LAT`, `DEMO_LON` | (unset) | If both set, `LocationSource` returns this point instead of asking GPS. Used for demos and simulator runs. |

Derived at runtime (not from env):

- `platform`: `dart:io` → `iOS` / `Android` / `macOS` / `Windows`
- `app_version`: read from `pubspec.yaml` via `package_info_plus`
- `timezone`: IANA name via `flutter_timezone`
- `timestamp`: hand-rolled ISO 8601 UTC second-precision string (Dart's
  `toIso8601String()` adds fractional seconds that fail the API's regex)

## Build & Run

### First-time setup

```bash
flutter create --platforms=android,ios,macos --org fi.ruohomaki --project-name nps_frontend .
flutter pub get
```

`flutter create` skips existing files, so `lib/`, `pubspec.yaml`, and the
hand-edited `macos/Runner/*.entitlements` are preserved.

### macOS (desktop)

```bash
flutter run -d macos \
  --dart-define=API_BASE_URL=http://localhost:8081/nps \
  --dart-define=API_KEY=dev-key \
  --dart-define=APP_ID=nps-frontend-demo
```

The macOS Debug + Release entitlements include
`com.apple.security.network.client` so the sandboxed app can make outbound
HTTP requests. Adding new entitlements means editing both
`macos/Runner/DebugProfile.entitlements` and
`macos/Runner/Release.entitlements`.

### iOS Simulator

```bash
# Boot an iOS sim (iOS 17.5 or newer)
xcrun simctl boot <UDID>
open -a Simulator

flutter run -d <UDID> \
  --dart-define=API_BASE_URL=http://localhost:8081/nps \
  --dart-define=API_KEY=dev-key
```

`localhost` resolves to the host on iOS simulators (unlike Android
emulators where you need `10.0.2.2`). On a real device replace with the
LAN IP of the machine running nps-api.

### Android

```bash
flutter run -d emulator-5554 \
  --dart-define=API_BASE_URL=http://10.0.2.2:8081/nps \
  --dart-define=API_KEY=dev-key
```

### Release builds

```bash
flutter build apk --release \
  --dart-define=API_BASE_URL=https://api.ruohomaki.fi/nps \
  --dart-define=API_KEY=<prod-key>

flutter build ios --release \
  --dart-define=API_BASE_URL=https://api.ruohomaki.fi/nps \
  --dart-define=API_KEY=<prod-key>
```

### Test

```bash
flutter test     # unit tests
flutter analyze  # static analysis (must be clean before commit)
```

## Theme System

`lib/theme.dart` defines `AppTheme.light()` plus a small palette of
named tokens. The look is *Fleet-inspired* (Boston's pattern library) —
Lora serif headings + Montserrat body, off-white surface, civic-clean
spacing — without copying any city's identity.

Palette tokens (use these, not raw `Color(0xFF...)` literals in screens):

| Token | Hex | Use |
|---|---|---|
| `AppTheme.primary` | `#1A4480` | Buttons, focused borders, brand mark |
| `AppTheme.primaryDark` | `#0F2E5C` | Secondary, hover/pressed |
| `AppTheme.surface` | `#FAFAF7` | Page background (warm off-white) |
| `AppTheme.surfaceElevated` | `#F0F0EC` | Subtle elevated sections |
| `AppTheme.onSurface` | `#1F1F1F` | Primary text |
| `AppTheme.onSurfaceMuted` | `#5A5A5A` | Secondary/label text |
| `AppTheme.border` | `#D9D9D6` | Card/input borders |
| `AppTheme.detractor` | `#C0392B` | NPS 1–6 (muted red) |
| `AppTheme.passive` | `#D4A535` | NPS 7–8 (amber) |
| `AppTheme.promoter` | `#2E8B5C` | NPS 9–10 (teal-green) |

`AppTheme.colorForRating(int rating)` returns the bucket color for a
1–10 score — use this when tinting rating chips, success-state icons,
or any UI tied to a rating value.

Typography: `GoogleFonts.lora*` for `displayLarge/.../titleLarge` and
`GoogleFonts.montserrat*` for everything else. Headlines lock in
`FontWeight.w600` so Lora renders with the intended weight.

## Logo Assets

`assets/images/` holds:

- `transparent-logo-512px.png` — used in the screen header
- `colored-logo-512px.png` — white-background variant for contexts where
  transparency is awkward (Dock icon previews, marketing thumbnails)

The screen displays the transparent variant at 160 px height; change the
`height:` value in `lib/screens/nps_screen.dart` (`_LogoSlot`) to adjust.

## Known Plugin Warnings

`flutter_timezone` emits a "does not support Swift Package Manager" warning
on every iOS/macOS build. Harmless under the current Flutter version; the
plugin still works via CocoaPods. Will become an error in a future Flutter
release, at which point either the plugin adopts SPM or we swap it.
